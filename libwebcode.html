Overview

The Library Management System is a Java Swing application designed to streamline library operations. It provides an intuitive graphical interface for managing books, users, transactions, and generating reports.

This application uses MySQL for data storage and incorporates the RS2XML (DbUtils) library for seamless database operations. The system follows a modular design with different sections for each library management function.

Technical Stack: Java, Swing, MySQL, JDBC, RS2XML (DbUtils)

Key Features

Book Management

Add, update, delete, and search books in the library collection. Maintains book ID and name information.

User Management

Manage library users with capabilities to add, update, and delete user records. Prevents deletion of users with active loans.

Transaction Handling

Issue and return books with automatic fine calculation for overdue items (7-day loan period with $1/day fine after due date).

Reporting System

Generate various reports including borrowing history, user-specific history, and book availability status. Export functionality to CSV.

User-Friendly Interface

Intuitive card-based navigation with a consistent layout. Real-time status updates with system date and time.

Database Integration

Robust MySQL integration with proper error handling and SQL injection prevention through prepared statements.

Setup Instructions

Prerequisites

  • Java Development Kit (JDK) 8 or higher
  • MySQL Server 5.7 or higher
  • MySQL Connector/J
  • RS2XML (DbUtils) library

Installation Steps

Install Java

Download and install the latest JDK from the official Oracle website. Set the JAVA_HOME environment variable.

Install MySQL

Download and install MySQL Server. Remember the root password you set during installation.

Download Libraries

Download MySQL Connector/J and RS2XML libraries. Add these JAR files to your project's classpath.

Database Setup

Create a new database named 'library' and execute the SQL script provided with the application to create the necessary tables.

Configure Connection

Update the database connection settings in the application to match your MySQL installation (host, port, username, password).

Run Application

Compile and run the Java application. The main class is typically named LibraryManagementSystem.java or similar.

Database Schema

The application uses a MySQL database with the following tables:

Books Table

Field Type Description
book_id INT (Primary Key) Unique identifier for each book
title VARCHAR(255) Title of the book
author VARCHAR(255) Author of the book
available BOOLEAN Availability status (true if available)

Users Table

Field Type Description
user_id INT (Primary Key) Unique identifier for each user
name VARCHAR(255) Full name of the user
email VARCHAR(255) Email address of the user

Transactions Table

Field Type Description
transaction_id INT (Primary Key) Unique identifier for each transaction
book_id INT (Foreign Key) Reference to the borrowed book
user_id INT (Foreign Key) Reference to the user who borrowed the book
issue_date DATE Date when the book was issued
due_date DATE Date when the book is due to be returned
return_date DATE Date when the book was returned (NULL if not returned)
fine_amount DECIMAL(10,2) Fine amount if returned after due date

Usage Guide

Adding Books

To add a new book to the library:

  1. Navigate to the 'Books' section
  2. Click the 'Add New Book' button
  3. Fill in the book details (title, author)
  4. Click 'Save' to add the book to the database

Figure 6: Add Book Interface

Managing Users

To add a new library user:

  1. Navigate to the 'Users' section
  2. Click the 'Add New User' button
  3. Enter the user's name and email address
  4. Click 'Save' to add the user to the database

Figure 7: Add User Interface

Issuing Books

To issue a book to a user:

  1. Navigate to the 'Transactions' section
  2. Click the 'Issue Book' button
  3. Select a user from the dropdown list
  4. Select an available book
  5. Click 'Issue' to complete the transaction

Figure 8: Issue Book Interface

Returning Books

To return a borrowed book:

  1. Navigate to the 'Transactions' section
  2. Click the 'Return Book' button
  3. Select the transaction from the list of active loans
  4. The system will calculate any applicable fines
  5. Click 'Return' to complete the process

Figure 9: Return Book Interface

Generating Reports

To generate reports:

  1. Navigate to the 'Reports' section
  2. Select the type of report you want to generate
  3. Set any filters or date ranges if needed
  4. Click 'Generate Report'
  5. Use the 'Export' button to save the report as CSV

Figure 10: Reports Interface
